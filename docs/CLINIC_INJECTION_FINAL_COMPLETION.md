## 2. Summary of fixes made

- **Dead code:** Removed unused `humanizeEnum` import from injection-tokens page.
- **Null safety:** Board expired tab uses `(board.expiredOrProblemToday ?? [])` so missing or null array does not throw.
- **Drawer audit:** Token drawer uses context first, then list row (`detailToken`), for Generated by / Validated by display so operator names show even when context fetch fails.
- **Open visit link:** Link to visit is shown only when `detailToken.visitId != null && Number(detailToken.visitId) > 0`, avoiding `/visits/undefined`.

No architecture or API changes. No new dependencies or migrations.

## 6. Final QA checklist

Use this for internal testing and pilot.

### Injection-tokens

- [ ] List loads with default filters; pagination (Previous/Next) when total > pageSize.
- [ ] Filters: status, operator (All / Validated by me / Generated by me), token code, from/to date; list and total update; page resets to 0 on filter change.
- [ ] Generate: visit search → select visit → optional prescription/course/day/vial → Generate; token appears in list; drawer shows Generated by (and Validated by when applicable).
- [ ] Open token drawer: View → drawer shows status, visit, medicine, dose, audit (Generated by, Validated by, Used/Cancelled when applicable). "Open visit" appears only when visit ID is present.
- [ ] Cancel: Cancel → modal with optional reason → confirm; token status CANCELLED; drawer closes if same token.

### Injection-room

- [ ] Board: Pending / Unassigned / Completed today / Bypass / Expired tabs show correct data; room filter and Validated by me / Administered by me filters work; no crash when a section is empty or missing.
- [ ] Validate: enter valid token code → success, step 2; re-validate same code → "already validated" message, can proceed.
- [ ] Steps 2–4: summary → source/vial (room mismatch warning if vial in different room) → record dose (expected dose and dose-differs warning when different).
- [ ] Record dose: with token and correct vial → success; board refreshes; token cleared.
- [ ] Room mismatch: token with pre-selected vial in room A, select vial in room B → warning in Step 3; submit → 400 with friendly message.
- [ ] Bypass: Emergency bypass → modal, reason required → submit; dose recorded; board bypass tab shows entry with Administered by when present.
- [ ] Expired tab: renders even when `expiredOrProblemToday` is empty or undefined (no throw).

### Cross-flow

- [ ] Generate on tokens page → validate and record dose on injection-room → token shows USED in list; drawer shows Used by and time.
- [ ] Permissions: without injection.token.cancel, Cancel not shown; without emergency_bypass, bypass button not shown; AccessDenied when missing required permission.
