# Clinic Injection Tokens & Injection Room — Phase 5 Gap Analysis

**Focus:** Monitoring, reporting visibility, final operational completion.  
**Date:** 2026-03-14.  
**Context:** Phase 4 complete (operator filters, unassigned bucket, room mismatch, dose consistency).

---

## 1. OPERATOR COMPLETION GAP

| Capability | Status | Notes |
|------------|--------|--------|
| My completed injections today | **Missing** | Board returns completedToday for the branch/date but has no filter by administeredByUserId. Operator cannot limit "Completed today" tab to doses they administered. |
| Administered by me filter | **Missing** | Same as above; no administeredByMe (or administeredByUserId) query param on board API. |
| Operator summary visibility | **Partial** | Each completed row shows "Administered by X"; no aggregate "injections per operator today" or summary block. |
| Per-operator workload review | **Missing** | No API or UI for "list operators with count of doses today" or workload-by-operator view. |

**Conclusion:** Add administeredByUserId filter to board for completedToday (and optionally bypassToday). Surface "Administered by me" checkbox on injection-room board. Per-operator summary can be a later enhancement.

## 2. MANAGER VISIBILITY GAP

| Item | API | UI (token list) | UI (board) |
|------|-----|------------------|------------|
| Generated by | Yes (list includes generatedBy) | **Missing** in table; only in token detail drawer | N/A (board is not token list) |
| Validated by | Yes (list includes validatedBy) | **Missing** in table; only in drawer | Yes (pending tokens show validatedBy badge) |
| Administered by | Yes (completedToday includes administeredBy) | N/A | Yes (completed list shows administered by) |
| Room | List does not include selectedVialSession/room | **Missing** | Yes (pending show room badge from selectedVialSession) |
| Problem / mismatch cases | expiredOrProblemToday returned; no "mismatch" list (mismatch prevented at record) | Token list can filter status=EXPIRED/CANCELLED | Yes (Expired / problem tab) |

**Conclusion:** Add optional "Generated by" and "Validated by" columns to the injection-tokens table (data already in list response). Keep list dense (e.g. short names or initials). Room on token list would require adding selectedVialSession to list response; optional for Phase 5 to avoid payload growth.

## 3. ROOM MODEL REVIEW

**Current:** Room is vial-derived only (VialSession.roomId; token has selectedVialSessionId; no roomId on InjectionToken or MedicationAdministration).

| Aspect | Stay vial-derived | Add roomId on token |
|--------|-------------------|----------------------|
| **Pros** | Single source of truth (vial location); no sync; already enforced (room mismatch at record); unassigned bucket covers "no room". | Explicit "token assigned to room" before vial selection; can show room in token list without joining vial. |
| **Cons** | Token without selected vial has no room; list "room" requires join to vial. | Duplication; risk of token room ≠ vial room; migration and backfill. |
| **Migration risk** | None | Add optional roomId to InjectionToken; backfill from selectedVialSession.roomId; update generation/assignment flows. |

**Recommendation:** **Keep vial-derived for Phase 5.** Current model is sufficient for room filtering, unassigned bucket, and room mismatch enforcement. Add token-level roomId only if a future requirement appears (e.g. "assign token to room before any vial is selected"). No migration in Phase 5.

## 5. IMPLEMENTATION PLAN (Phase 5)

**A. Backend/API**  
1. **Board:** Add optional administeredByUserId to getInjectionRoomBoard. When set, filter completedToday (and bypassToday) by administeredByUserId. Controller: read administeredByMe (and administeredByUserId), resolve "me" to req.user.id, pass to service.  
2. **Board bypass list:** Include administeredBy in bypassToday select for consistency with completedToday.  
3. **List tokens:** No change (already returns generatedBy, validatedBy). Optional: add selectedVialSession with room to list for "room" column; prefer Phase 5 without to keep list lean.

**B. Prisma/migration**  
None.

**C. Frontend**  
1. **Injection-room board:** Add "Administered by me" checkbox; pass administeredByMe to board API; when checked, completed (and bypass) tabs show only current user's doses.  
2. **Injection-tokens list:** Add "Generated by" and "Validated by" columns to the table (from list response); keep compact (e.g. displayName or initials).  
3. **Bypass tab:** When backend returns administeredBy for bypass, show "Administered by X" in bypass list (same pattern as completed).  
4. **Reporting polish:** No new tabs; ensure existing board tabs and token list remain readable and consistent.

**D. QA**  
- Board: "Administered by me" filters completed (and bypass) to current user.  
- Token list: Table shows Generated by and Validated by.  
- Bypass list shows administered by when provided.  
- No regression on list/board when filters not used.
