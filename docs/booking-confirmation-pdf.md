# Booking Confirmation PDF

**Date:** 2026-06-07  
**Endpoint:** `GET /api/v1/campaign/bookings/:reference/pdf`

---

## Overview

Server-generated A4 booking confirmation PDF using **pdfkit** + **qrcode**. Data is loaded from `campaign_bookings` and related tables — not from client-only state.

---

## API

### Request

```
GET /api/v1/campaign/bookings/{bookingRef}/pdf?code={verificationCode}
```

| Param | Location | Required | Description |
|-------|----------|----------|-------------|
| `reference` | path | Yes | Booking ref (e.g. `VAC-ABC123`) |
| `code` | query | Yes* | Verification code (with or without hyphen) |
| `verificationCode` | query | Alt | Alias for `code` |

\* Not required when caller is authenticated as `ownerUserId` on the booking.

### Response

- `200` — `application/pdf` attachment  
- `401` — Invalid verification / not owner  
- `404` — Booking not found  
- `429` — Rate limited  

**Filename:** `BPA-Booking-{BOOKING_REF}.pdf` (e.g. `BPA-Booking-VAC-68PRKA.pdf`)

---

## Security

1. Booking must exist (`bookingRef`).
2. Access granted if **either**:
   - Query `code` matches `generateVerificationCode(qrToken)` (normalized), or
   - JWT user `id` equals `booking.ownerUserId`.
3. Rate limit: 10 PDF requests / 15 min per `ref:ip` (in-memory).
4. No payment re-check required — PDF is a confirmation record, not a payment gate.

## PDF structure

| Section | Fields |
|---------|--------|
| Header | BPA logo block, “Bangladesh Pet Association (BPA)” |
| Campaign | Campaign name |
| Booking | Booking ID, Verification code |
| Customer | Name (mobile if Guest), Mobile |
| Pet(s) | Name, Species, Breed, Gender per pet |
| Vaccination | Campaign, Location/area, Venue, Date/time or “Will be sent via SMS” |
| Payment | Status, Method, Amount |
| QR | Multi-line: booking ID, verification code, verify URL (`…/verify/certificate?ref=&code=`) |
| Contact | Website, email, phone, address (see `bookingPdf.constants.ts`) |
| Footer | Generated by BPA Vaccination System, timestamp |

## Frontend integration

| Page | Behavior |
|------|----------|
| `/book/success` | Auto-download once per booking (`sessionStorage` key `bpa_booking_pdf_auto_{ref}`); manual Download button |
| `/booking/[ref]` | Download button using stored verification code |

---

## Deployment

1. Deploy **backend-api** (pdfkit/qrcode already in `package.json`).
2. Deploy **vaccination_2026** (API download client).
3. Set `CAMPAIGN_LANDING_URL` for verification URL in PDF (defaults to production host).
4. Smoke: fulfilled checkout → success page auto PDF → refresh (no duplicate) → My booking download.
